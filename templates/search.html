The Python script search_engine.py applies latent semantic indexing and singular value decomposition to implement a toy search engine. To run the script, you'll need Python 2.6 or greater and the numpy library. Also the Pickle file lsiData.pkl must be in the same directory as the script or the path to a similarly encoded word-document count file must be provided on the command line. See the README for details.

The most important parameter for determining the behavior of the search engine specifies the number of components (k) to be used in truncated svd. Large values will result in more literal searches with little clustering of concepts. Small values will result in large clusters with few distinctions among keywords. As the scree plot below suggests an ideal choice for the number of components for the default data set is 10.
